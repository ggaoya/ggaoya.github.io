HTML代码规范

2017-08-22 • 标签: 代码优化
1.代码风格
1.1缩紧与换行
[强制]使用4个空格作为一个缩进层级，不允许使用2个空格或tab字符
1.2命名
[强制]class必须单词字母全小写，单词以-分割
[强制]class必须代表相应模块或部件的内容或功能，不得以样式信息命名
[强制]元素id必须保证页面唯一
[强制]禁止为了hook脚本，无样式信息的class

解释：不允许class只用于让javascript选择某些元素，class应具有明确的语意和样式，否则容易导致css class泛滥

[强制]同一页面应避免使用相同的name与id

解释：IE浏览器会混淆元素的id与name属性，document.getElementById可能获得到不期望的元素值

[建议]id单词全字母小写，单词以_分隔，同项目必须保持风格一致
[建议]id、class命名，在避免冲突并描述清楚的前提下尽可能缩短
1.3标签
[强制]标签名必须小写
[强制]对于无需自闭合标签，禁止强制自闭合

例如：input、br、img

[建议]在css可以实现相同需求的情况下不可使用表格进行布局
[建议]标签的使用尽量简洁化，减少不必要的嵌套
1.4属性
[强制]属性名必须使用小写字母，自定义属性视情况而定
[强制]属性值必须使用双引号包围，自定义属性视情况而定
[建议]布尔类型的属性，建议不添加属性值
[建议]自定义属性建议以XXX-为前缀，推荐使用data-
2.Head
2.1 DOCTYPE
[强制]使用HTML5的doctype来启用标准模式，建议使用大写的DOCTYPE
[建议]启用IE Edge模式
[建议]在html标签上设置正确的lang属性

<html lang="zh-CN">

2.2编码
[强制]页面必须使用精简形式，明确指定字符编码，指定的字符编码必须是head的第一个直接子元素
2.3title
[强制]页面必须包含title标签声明标题
[强制]title必须作为head的直接子元素，并紧随charset 声明之后
2.4favicon
[强制]保证favicon可访问
2.5css和javascript引入
[强制]引入css时必须指明rel=”styleheet”
[建议]引入css和javascript时无需指明type属性
[建议]在head中引入页面需要的所有css资源
[建议]javascript应当放在页面末尾，或采取异步加载
2.6viewport
[建议]若页面欲对移动设备友好，需指定页面的viewport
2.7IE Style Fixed
[建议]为兼容IE9及以下浏览器,有必要在head中使用css if IE条件注释
3.图片
[强制]禁止img的src取值为空。延迟加载的图片也要设置默认的src
[建议]避免为img添加不必要的title属性
[建议]为重要的图片添加alt属性
[建议]添加width和height属性，以避免页面抖动
[建议]有下载需求的图片采用img标签实现，无需下载的图片采用css背景图实现
4.表单
4.1控件标题
[强制]有文本标题的控件必须使用label标签将其与其标题相关联
4.2按钮
[强制]使用button元素时必须指明type属性值
[建议]尽量不要使用按钮元素的name属性
4.3可访问性
[建议]负责主要功能的按钮在DOM中的顺序应靠前
[建议]当使用javascript进行表单提交时，如果条件允许，应使原生提交功能正常工作
[建议]在针对移动设备开发的页面时，根据内容类型指定输入框的type属性
5.多媒体
[建议]当在现代浏览器中使用audio以及video标签来播放音频视频时，应当注意格式
[建议]在支持HTML5的浏览器中优先使用audio和video标签来定义音频视频元素
[建议]只有在必要的时候开启音视频的自动播放
[建议]在object标签内部提供指示浏览器不支持该标签的说明
6.模版中的HTML
[建议]模版代码的缩进优先保证HTML代码的缩进规则
[建议]模版代码应以保证HTML单个标签语法的正确性为基本原则
